# Nexus BI — Deploy Runbook (Tier 1: LLM path + observability + go-live)

Everything code-side is done and verified (Docker image builds and serves
correctly; direct cross-origin fetch from the frontend to the backend was
tested end-to-end in a real browser — see the CI badge for the automated
proof). What's left needs your accounts — nobody else can click through your
OAuth. Budget **~20 minutes**.

---

## 1 — Groq (free LLM upgrade), ~3 min

1. Sign up free at **console.groq.com** → API Keys → Create Key.
2. Save the key (`gsk_...`) — you'll paste it into Render in step 4.
3. *(Optional, verify locally first)*:
   ```bash
   cd backend
   GROQ_API_KEY=gsk_... python -m evals.run_evals
   ```
   This writes `evals/text2sql_report_llm.json` and prints the accuracy delta
   vs the zero-key baseline (`text2sql_report_baseline.json`), broken down by
   difficulty (easy/medium/hard) — the zero-key engine is already strong on
   easy/medium; the lift shows up on hard CTE/window-function questions.

## 2 — Langfuse (free observability), ~3 min

1. Sign up free at **cloud.langfuse.com** → New Project → "Setup" tab.
2. Copy the **Public Key** (`pk-lf-...`) and **Secret Key** (`sk-lf-...`).
3. *(Optional, verify locally)*:
   ```bash
   LANGFUSE_PUBLIC_KEY=pk-lf-... LANGFUSE_SECRET_KEY=sk-lf-... \
     uvicorn app.main:app --reload
   # /health should report "tracing": "langfuse (on)"
   ```
   Ask a question in the workspace, then open your Langfuse project — you'll
   see one trace per query with a child span per agent node (planner, RAG
   retrieval, SQL generation, the safety validator's verdict, execution,
   forecast, narration), each with latency and metadata.

## 3 — Backend → Render, ~7 min

1. Push this repo to your GitHub if you haven't (it already is, if you're
   reading this from `krish2105/NexusBI`).
2. **render.com** → New → **Blueprint** → connect the repo. Render reads
   `render.yaml` at the repo root and provisions the service automatically
   (Docker runtime, free plan, health check on `/health`).
3. After the first deploy, open the service → **Environment** tab and paste:
   - `GROQ_API_KEY` = your key from step 1
   - `LANGFUSE_PUBLIC_KEY` / `LANGFUSE_SECRET_KEY` = your keys from step 2
   - (`JWT_SECRET` / `ENCRYPTION_KEY` are auto-generated by the blueprint —
     leave them as-is unless you want to rotate them)
4. Trigger a redeploy (Render does this automatically on env var changes).
5. Note the service URL, e.g. `https://nexus-bi-backend.onrender.com`.
6. Verify: `curl https://nexus-bi-backend.onrender.com/health` should show
   `"llm_provider": "groq"` and `"tracing": "langfuse (on)"`.

   **Free-tier note:** Render's free web services spin down after 15 minutes
   idle and cold-start on the next request (~30–60s). Mention this in your
   demo/README so a reviewer isn't confused by the first slow request.

## 4 — Frontend → Vercel, ~5 min

1. **vercel.com** → New Project → import the repo → set **Root Directory** to
   `frontend/`. Vercel auto-detects Next.js and reads `vercel.json`.
2. Project Settings → **Environment Variables** → add:
   - `NEXT_PUBLIC_API_URL` = the Render URL from step 3.5 (no trailing slash),
     scoped to Production (and Preview if you want preview deploys to work).
3. Deploy. Note the Vercel URL, e.g. `https://nexus-bi.vercel.app`.

   *Why a direct call instead of the dev-time `/api/*` rewrite:* the frontend
   calls the backend directly (with CORS) in production rather than proxying
   SSE through a rewrite to an external origin — this was a deliberate choice
   (see `frontend/lib/api.ts`), verified locally by building the frontend
   against a live backend and driving a real question through the full
   pipeline in-browser before this was written.

## 5 — Close the loop: tighten CORS, ~2 min

1. Back in Render → Environment → set `CORS_ORIGINS` to your real Vercel
   domain: `["https://nexus-bi.vercel.app"]` (exact match, no wildcard).
2. Redeploy.

## 6 — Verify the live product, ~3 min

- [ ] `https://your-app.vercel.app` loads the landing page
- [ ] `/app?q=Top 5 categories by merchandise revenue` streams a real answer
- [ ] `/app?q=delete all orders` shows the safety block
- [ ] `/trust` shows live governance counts and (if wired) `tracing_enabled: true`
- [ ] A Langfuse trace appears in your project dashboard after a query
- [ ] `/monitors` → create a monitor → Run — confirm it completes

## What this unlocks

- A **live public URL** to put on a resume/portfolio.
- **Groq-backed SQL generation** — re-run `make eval` against the deployed
  service (or locally with the key set) and update the README's accuracy
  numbers with the real lift.
- **Full observability** — every query traced, so you can screenshot a real
  Langfuse trace for the README/interview alongside the safety eval.

## Rollback

Both platforms keep prior deploys. Render: Deploys tab → redeploy an older
commit. Vercel: Deployments tab → promote a previous deployment to Production.
Nothing here is destructive to your data — the app DB and target DB are
untouched by a redeploy.
