# GTM — the Phase 2 validation kit

*[ROADMAP.md](../ROADMAP.md) Phase 2 is **conversations, not commits.** Its job is to
answer the one question that decides whether Phases 3+ are worth writing: is provable
safety a real buying criterion, or a checkbox a read-only role already ticks? This
folder is the kit to run that validation and record the answer.*

| File | What it's for |
|---|---|
| [`ICP.md`](./ICP.md) | The **one** named customer — mid-market Postgres/MySQL data teams, sold on governed access + audit. Positioning statement, where to find 5 of them. |
| [`OUTREACH.md`](./OUTREACH.md) | Outreach templates (cold email / LinkedIn / warm intro) + a 30-minute Mom-Test conversation script that asks *what they'd pay*, not *whether they like it*. |
| [`VALIDATION_VERDICT.md`](./VALIDATION_VERDICT.md) | The deliverable: a log for the 5 conversations, the scoring tally, the pre-committed decision gate, and the verdict to write down and commit. |

**The loop:** name the ICP → run 5 conversations → tally → apply the gate → commit the
verdict. If the honest answer is "the wedge is a checkbox," that's a *successful*
validation — it saves months of building the wrong thing and the project stays a
top-of-portfolio hiring artifact ([CASE_STUDY.md](../CASE_STUDY.md)), which was always
the higher-probability payoff.
